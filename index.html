Three Important Questions

How does Javascript work?

Ans: JavaScript is a single threaded programming language.Client browser interprete JavaScript at runtime. It is also known as dynamic language.It has a single call stack and can do one thing at a time. JavaScript codes are executed by Chrome V8 engine. This engine is built with two major components, they are Memory heap and Call stack. When we run a JavaScript code inside a browser, the browser's V8 engine receives the code and the output is obtained whenever the engine runs the code.

What does Javascript's event loop do?

Ans:Event loop is an important thing where JavaScript's run time model is based on. Event loop is responsible for processing events, code execution and queued sub-task execution. Event loop runs constantly. It monitors the call back queue and call stack. The event loop waits until the call stack is empty then it places the upcoming function from the callback queue to the call stack.

What is the difference between local storage and session storage?

Ans:The local storage and session storage both are special objects built for storing data. The main difference between local storage and session storage is the data in local storage doesn't expire while the data in session storage expires after a session breaks. SessionStorage is considered a better choice over localStorage because when browser tabs are closed session data is cleared. Session storage is designed to be temporary. On the other hand local storage solves the "Login going away issues". Session storage deals with changes made which are saved and available for the current page
